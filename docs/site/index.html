The language for AI agent pipelines

Build, test, and debug AI agent workflows in a fraction of the code. Harn has native LLM calls, tool use, structured output, and async concurrency built into the language.

Get Started View on GitHub 
pipeline default(task) {
  let tools = tool_registry()
    |> tool_add("search", "Search the web", search_fn, {query: "string"})

  let result = llm_call(task, "You are a research assistant", {
    tools: tools,
    response_format: "json",
  })

  log(result.data)
}

Quickstart

1. Install

cargo install harn-cli

Or build from source:

git clone https://github.com/burin-labs/harn
cd harn && cargo build --release
cp target/release/harn ~/.local/bin/

2. Create a project

harn init my-agent
cd my-agent

3. Write your first pipeline

// main.harn
pipeline default(task) {
  let response = llm_call(
    "Summarize: ${task}",
    "Be concise."
  )
  log(response)
}

4. Run it

export ANTHROPIC_API_KEY=sk-...
harn run main.harn

Why Harn?

Native LLM Calls

Call Claude, GPT, or local models with a single function. Structured JSON output, streaming, and multi-turn conversations built in.

let data = llm_call(prompt, system, {
  response_format: "json",
  model: "claude-sonnet-4-20250514",
})
log(data.data.summary)

Tool Use

Register functions as tools and let the LLM call them. Uses native Anthropic tool_use and OpenAI function calling protocols.

let tools = tool_registry()
  |> tool_add("read_file", "Read a file",
     read_fn, {path: "string"})

let r = llm_call(task, sys, {tools: tools})

Pipeline Operator

Chain operations with |> for readable data transformation pipelines.

let result = data
  |> filter(fn(x) { x.score > 0.8 })
  |> map(fn(x) { x.name })
  |> join(", ")

Async Concurrency

Run tasks in parallel with spawn/await, channels, mutexes, and atomics. Deadlines and timeouts built in.

parallel(3) { i ->
  let result = llm_call(tasks[i])
  send(ch, result)
}

Execution Tracing

See every LLM call, token count, and cost with --trace. Record and replay LLM responses for deterministic CI testing.

$ harn run --trace agent.harn
# --- LLM trace ---
#  #1: claude-sonnet | 1.2k in + 350 out | 1200ms
#  1 call, 1550 tokens, ~$0.0089

First-class Testing

Test pipelines with test_* naming convention, assertion builtins with source locations, auto-discovery, and watch mode.

pipeline test_extraction(task) {
  let r = llm_call(prompt, sys, {
    response_format: "json"
  })
  assert_eq(r.data.name, "Alice")
}

Language Guide

Basics

  • Pipelines & Functions
  • Variables: let / var
  • Types: int, float, string, bool, list, dict
  • Control flow: if, for, while, match
  • Pattern matching with enums
  • Try / catch / throw

AI Agent Features

  • LLM calls: llm_call, llm_stream
  • Agent loops: agent_loop with tool dispatch
  • MCP: tools, resources, and prompts
  • Tool registries with JSON Schema
  • Tracing, timers, & cost tracking
  • Project metadata store

Concurrency

  • spawn / await / cancel
  • parallel / parallel_map
  • Channels: channel, send, receive
  • Mutexes & atomics
  • Deadlines & timeouts
